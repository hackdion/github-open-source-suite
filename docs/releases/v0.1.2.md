# v0.1.2 Release Notes

## 中文

### 摘要

`v0.1.2` 是第二轮多仓库 dogfood 驱动的精修版本。目标不是扩展技能数量，而是把“仓库类型”和“仓库 readiness”分开判断，避免把“本地可用”误判成“已适合开源发布”。

### 主要改进

- 新增 readiness 视角：共享分类指南现在明确区分 `Local-only`、`Repo-foundation-ready`、`Community-surface-ready`、`Publish-ready`、`Release-ready`
- 强化 `oss-repo-audit`：现在会把“无 commit 的仓库”直接回退到 `oss-repo-bootstrap`，并在审计结果里显式输出 readiness 状态
- 强化 `oss-community-files`：对早期或单技能仓库优先生成最小可持续的公开表面，不再默认把部分文件存在视作社区基线已完成
- 强化 `oss-publish-github`：明确“本地测试通过”“本地 V1 已完成”或“已有 README/LICENSE”都不是发布就绪证明
- 强化 `oss-release-management`：禁止把仅本地完成的里程碑直接包装成正式 release 准备
- 新增第二轮 dogfood 报告：`docs/dogfood/2026-04-10-multi-repo-round-2-friction-report.md`

### 已知限制

- GitHub 设置自动化仍依赖登录态、仓库权限和平台可操作性
- 这套技能仍主要聚焦 GitHub 开源发布，不替代目标仓库自身的深度文档设计或产品文档治理

## English

### Summary

`v0.1.2` is a second-round, multi-repo dogfood refinement release. Its goal is to separate repository type from repository readiness so the suite stops overreading "locally usable" as "ready for open-source publication".

### Highlights

- Added explicit readiness states to the shared classification guide: `Local-only`, `Repo-foundation-ready`, `Community-surface-ready`, `Publish-ready`, and `Release-ready`
- `oss-repo-audit`: now routes no-commit repositories back to `oss-repo-bootstrap` and reports readiness state explicitly
- `oss-community-files`: now prefers the minimum sustainable public surface for early or single-skill repositories
- `oss-publish-github`: now treats local test success, internal milestones, or partial root docs as insufficient publication signals
- `oss-release-management`: now blocks premature release framing when the repo is only locally complete
- Added second-round dogfood report at `docs/dogfood/2026-04-10-multi-repo-round-2-friction-report.md`

### Known Limitations

- GitHub settings automation still depends on active auth, repository permissions, and platform operability
- The suite focuses on GitHub open-source publication and does not replace deeper documentation design work inside target repositories
