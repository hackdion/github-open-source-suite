# v0.1.1 Release Notes

## 中文

### 摘要

`v0.1.1` 是一次 dogfood 驱动的精修版本。目标是提升在“已有结构的真实仓库”上的判断准确性，而不是扩展技能数量或增加复杂度。

### 主要改进

- 强化 `oss-repo-audit`：明确把“安全上报路径不足”标记为策略风险，而不是仅因为 `SECURITY.md` 存在就给出过高评估
- 强化 `oss-publish-github`：明确区分“内容一致但历史分叉”和“内容差异”，避免误把历史形态当作发布阻塞
- 强化 `oss-release-management`：增加“发布文档应保持可复用，不要写成一次性 runbook”的规则
- 新增 dogfood 报告：`docs/dogfood/2026-04-10-opencode-for-codex-skill-friction-report.md`

### 已知限制

- 是否可自动完成 GitHub 设置仍依赖登录态与权限
- 外部依赖健康状态（例如上游 API 稳定性）仍属于仓库外部风险

## English

### Summary

`v0.1.1` is a dogfood-driven refinement release. It improves decision quality on already-structured real repositories rather than adding new skills.

### Highlights

- `oss-repo-audit`: now classifies weak security reporting routes as policy risk instead of over-scoring readiness based only on file presence
- `oss-publish-github`: now separates content parity from git-history divergence to avoid false publication blockers
- `oss-release-management`: now enforces durable release guidance and avoids one-time runbook style in long-lived docs
- Added dogfood report at `docs/dogfood/2026-04-10-opencode-for-codex-skill-friction-report.md`

### Known Limitations

- GitHub setting automation still depends on active auth and repository permissions
- External provider stability remains an out-of-repo risk
