# v0.1.0 Draft Release Notes

## 中文

### 摘要

这是 `开源到 GitHub 的标准流程技能` 的首个公开版本草稿。它把“开发接近完成后的 GitHub 开源准备、发布、首发版和后续维护”整理成一套可复用的 Codex 技能。

### 主要内容

- 新建 6 个核心技能，覆盖仓库初始化、开源体检、社区文件生成、GitHub 发布准备、release 管理和维护者运营
- 新建共享规范与模板层，减少每次开源都要重新整理流程和文件
- 为当前仓库自身补齐开源所需的根级文件和 `.github` 模板
- 把“一次性收集前置确认项，再执行”正式写入技能逻辑

### 已知限制

- 当前版本仍然需要在 GitHub 上完成仓库创建、Discussions 启用和 private vulnerability reporting 启用
- 当前仓库还没有首个 git commit
- 某些 GitHub 设置是否能由 AI 直接代操作，仍取决于实时登录态与权限

## English

### Summary

This is the draft first public release of `GitHub Open Source Suite`, a Codex skill suite for turning nearly finished local work into GitHub-ready open-source repositories and supporting first-release plus long-term maintenance.

### Highlights

- six core skills for repository bootstrap, audit, community files, publication prep, release management, and maintainer operations
- shared references and templates to avoid rebuilding the open-source publishing process from scratch each time
- root repository community-health files and `.github` templates for this repository itself
- an explicit one-shot upfront-input collection model so execution does not stop for scattered follow-up questions

### Known Limitations

- GitHub-side repository creation, Discussions enablement, and private vulnerability reporting still require remote setup
- the repository does not yet have its first git commit
- browser-based GitHub settings automation still depends on real-time login state and permissions
