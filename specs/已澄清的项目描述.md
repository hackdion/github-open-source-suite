# 已澄清的项目描述

**项目名称**：开源到 GitHub 的标准流程技能 / GitHub Open Source Suite  
**核心价值**：把“本地项目接近完成后，如何规范地开源到 GitHub 并持续维护”的重复劳动沉淀成一套可复用技能。  
**目标用户**：以 Codex 为代表的 AI 编程代理，以及需要把技能、MCP、插件、库、CLI、小中型项目发布到 GitHub 的独立开发者。  
**关键特性**：
- 基于仓库状态和类型，判断当前应走 bootstrap、audit、community files、publish、release 还是 maintainer 阶段
- 提供 GitHub 社区健康文件、发布说明和维护规则的共享规范与模板
- 通过真实仓库 dogfood 反向收敛技能边界，减少“本地可用”和“可公开发布”之间的误判
**约束条件**：
- 以 GitHub 官方文档与开源维护最佳实践为基线
- 优先自动执行低风险本地动作，高影响 GitHub 写入必须明确验证
- 输出要支持中英双语文档场景
