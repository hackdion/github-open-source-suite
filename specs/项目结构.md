# 项目结构

## 1. 目录树 (Directory Tree)

```text
/
├── specs/                         # SpecForge 项目核心定义
├── docs/
│   ├── dogfood/                   # 真实仓库验证报告
│   ├── releases/                  # release notes
│   └── superpowers/
│       ├── specs/                 # 历史设计文档
│       └── plans/                 # 历史实施计划
├── shared/
│   ├── references/               # 跨技能共享规范
│   └── assets/                   # 跨技能共享模板
├── oss-repo-bootstrap/           # Git 初始化与基础 readiness
├── oss-repo-audit/               # 开源 readiness 审计
├── oss-community-files/          # 社区健康文件与模板
├── oss-publish-github/           # GitHub 发布准备
├── oss-release-management/       # release 管理
├── oss-maintainer-operations/    # 维护规则
├── .github/                      # issue / PR 模板
├── README.md
├── CHANGELOG.md
├── SECURITY.md
├── SUPPORT.md
├── CONTRIBUTING.md
├── CODE_OF_CONDUCT.md
└── LICENSE
```

## 2. 关键文件说明 (Key Files)

- **`README.md`**: 仓库入口文档，说明项目目标、工作流和默认配置。
- **`CHANGELOG.md`**: 版本级变更记录。
- **`specs/`**: SpecForge V3 项目治理资产，是后续项目级 Skill 的优先输入。
- **`docs/dogfood/`**: 真实仓库验证结论，用于反向修正技能边界。
- **`docs/releases/`**: 已发布版本和 draft 的 release 文案。
- **`shared/references/`**: 仓库分类、发布输入、维护规则等共享规范。
- **`shared/assets/`**: 社区文件、release notes、维护流程模板。

## 3. 模块说明 (Module Description)

- **`oss-repo-bootstrap/`**: 负责把本地目录转成可进入开源流程的 Git 仓库。
- **`oss-repo-audit/`**: 负责识别仓库类型、readiness 阶段、缺口与 handoff。
- **`oss-community-files/`**: 负责生成和补齐 GitHub 社区健康文件。
- **`oss-publish-github/`**: 负责 GitHub 发布准备、元信息与高影响动作前置检查。
- **`oss-release-management/`**: 负责版本 framing、release notes 与发版门禁。
- **`oss-maintainer-operations/`**: 负责 issue/PR/support/security 的维护策略。
- **`shared/references/`**: 只放共享规则，不放项目个性化叙述。
- **`shared/assets/`**: 只放复用模板，不放项目治理文档。

## 4. 文件放置规则 (Placement Rules)

- **项目级治理文档**：放在 `specs/`
- **设计与实施历史文档**：放在 `docs/superpowers/specs/` 与 `docs/superpowers/plans/`
- **真实 dogfood 结论**：放在 `docs/dogfood/`
- **release 文档**：放在 `docs/releases/`
- **技能定义**：每个技能单独放在顶层 `oss-*` 目录，目录内至少包含 `SKILL.md` 与 `agents/openai.yaml`
- **共享规则**：放在 `shared/references/`，禁止复制到多个技能里维护
- **共享模板**：放在 `shared/assets/`，技能只引用，不在各技能目录重复存放
- **根级开源文件**：保留在仓库根目录，供 GitHub 直接识别
