# 产品概述

## 1. 项目愿景 (Vision)

为已经开发得差不多的本地成果，提供一套可复用、可验证、可持续的 GitHub 开源发布标准流程。

## 2. 核心价值 (Value Proposition)

- **用户价值**：减少每次开源前重新梳理 README、LICENSE、SECURITY、release、支持渠道和维护规则的重复劳动。
- **商业/技术价值**：把 GitHub 开源发布实践沉淀成稳定技能，降低 AI 代理在不同仓库上重复探索和误判的成本。

## 3. 核心板块 (Core Modules)

### 3.1 仓库基础与 readiness 判断

- **描述**：负责把本地目录转成可管理仓库，并判断当前所处的 readiness 阶段。
- **关键特性**：
  - `github-open-source-suite`
  - `oss-repo-bootstrap`
  - `oss-repo-audit`

### 3.2 社区健康文件与公开表面

- **描述**：负责生成或补齐 GitHub 识别的社区健康文件与模板。
- **关键特性**：
  - `oss-community-files`
  - `shared/assets/community/*`

### 3.3 GitHub 发布与 release 管理

- **描述**：负责 GitHub 仓库元信息、发布前检查、release framing 与 release notes。
- **关键特性**：
  - `oss-publish-github`
  - `oss-release-management`

### 3.4 维护与共享规范

- **描述**：负责发布后的 issue/PR/support/security 维护规则，以及共享规范的持续演进。
- **关键特性**：
  - `oss-maintainer-operations`
  - `shared/references/*`
  - `docs/dogfood/*`

## 4. 目标用户 & 核心场景 (User & Scenes)

### 4.1 目标用户

- **独立开发者 / 维护者**：已经把项目做得差不多，但不想每次手工重建 GitHub 开源流程。
- **AI 编程代理**：需要在已有项目事实基础上，稳定判断“现在该补什么、该问什么、该停在哪一步”。

### 4.2 核心场景 (Key Use Cases)

- **场景 1**：维护者把一个本地技能或 MCP 做完后，希望 AI 判断它是否已经具备开源条件，并补齐缺失的社区文件。
- **场景 2**：仓库已经有基本结构，但不确定下一步是发版、开源、还是先补支持与安全路径。
- **场景 3**：维护者希望在公开发布后，继续用统一规则管理 issue、PR、support 和 security。

## 5. 关键业务流程 (Core Workflows)

### 流程一：从本地成果到 GitHub 开源发布

本地目录 -> `oss-repo-bootstrap` -> `oss-repo-audit` -> `oss-community-files` -> `oss-publish-github` -> `oss-release-management`

### 流程二：从真实仓库反馈回收束技能

真实仓库只读 dogfood -> friction report -> 收敛 readiness / handoff / template 规则 -> 迭代技能 -> 新版本发布

## 6. 验收标准 (Acceptance Criteria)

- **功能完备性**：
  - 6 个核心技能都存在并可触发
  - 共享规范与模板可复用
  - 当前仓库自身可作为开源发布样本
  - 至少经过多轮真实仓库 dogfood
- **技术指标**：
  - 关键技能通过结构校验
  - release 文档、dogfood 报告、README 和共享规范之间不互相矛盾
  - 高影响 GitHub 动作有写前摘要和写后回读验证

## 7. 约束条件 (Constraints)

- 必须以 GitHub 官方开源与社区健康基线为主
- 默认只自动执行低风险本地动作与低风险 GitHub 草拟动作
- 文档需兼容中文与英文场景
- 不能把不属于 GitHub 开源发布的问题都硬塞进当前项目范围

## 8. 项目边界 (Out of Scope)

- 不替目标仓库实现其业务功能
- 不替目标仓库编写完整业务文档体系
- 不覆盖 GitHub 之外平台的完整发布流程
- 不承诺自动完成所有浏览器型 GitHub 设置；有权限或平台限制时允许退回为精确指引
