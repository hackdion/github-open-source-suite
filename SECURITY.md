# Security Policy / 安全策略

## 中文

本仓库公开后，维护者计划优先启用 GitHub 的私密漏洞报告入口。

推荐流程：

1. 仓库公开后，如果你在仓库界面看到了 GitHub 的私密漏洞报告入口，请优先使用该入口
2. 不要把漏洞细节直接发到公开 issue 或 discussion
3. 提交时请尽量包含受影响范围、复现方式、影响说明和可能的缓解方案

如果仓库已经公开，但你暂时还没有看到私密漏洞报告入口：

- 不要公开披露漏洞细节
- 可以先开一个不包含敏感技术细节的 GitHub Discussion，请维护者建立私密沟通路径

## English

After this repository is published publicly, maintainers intend to use GitHub private vulnerability reporting as the preferred intake path.

Recommended process:

1. once the public repository shows the private vulnerability reporting entry, use that path first
2. do not post vulnerability details in a normal public issue or discussion
3. include affected scope, reproduction details, impact summary, and any known mitigations when possible

If the repository is already public but you do not yet see a private reporting entry:

- do not disclose technical details publicly
- open a GitHub Discussion without sensitive details and ask maintainers to establish a private reporting path

## Response Expectations

Security review is best-effort and response times may vary.
